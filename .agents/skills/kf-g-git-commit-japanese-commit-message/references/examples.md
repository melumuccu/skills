# 出力例

## 例 1

```text
fix__: 認証_ログイン失敗時のエラーメッセージ表示を修正

- 概要: 認証失敗時にエラートーストを表示
- Why: 失敗理由が分からず再試行しづらい状態の解消
```

## 例 2

```text
docs_: README_ローカル開発手順を追記

- 概要:
  - 初回セットアップ手順を追記
  - 起動確認手順を追記
- Why:
  - 新規参加者が環境構築で詰まりやすい状況
  - 参照先分散の解消
```

## 例 3

```text
feat_: 動画カード_スマホ長押し操作メニューを追加

- 概要:
  - タッチ長押しで動画カードの操作ポップアップを表示
  - 長押し後の誤タップ抑止と関連テストを追加
- Why:
  - スマホでは右クリック導線が存在しないため
  - モバイルでも PC と同じ操作を使えるようにするため
```

## 例 4

```text
chore: skills_コミットメッセージ skill の判定ルール追記

- 概要:
  - skill の判断基準に docs repo と `feat_` の制約を追記
  - subject のカテゴリ付与ルールを追記
- Why:
  - AI agent 向けファイル変更はプロジェクト動作へ直接影響しないため
  - commit message の分類基準を明確にするため
```

## 例 5

```text
refa_: オンボーディング_セットアップ導線の章構成整理

- 概要:
  - 初回セットアップの章立てを再編
  - 重複していた前提条件を統合
- Why:
  - docs repo では文書更新自体が自明で `docs_` では粒度が荒すぎるため
  - 情報設計の整理を commit message から判別しやすくするため
```

## 完成形のサンプル

```text
feat_: 招待管理_ユーザー招待メール再送機能を追加

- 概要:
  - 管理画面に再送操作を追加
  - 招待メール再送 API を追加
- Why:
  - 期限切れや未達時の再招待手段が必要
  - 手動対応負荷の軽減
```
